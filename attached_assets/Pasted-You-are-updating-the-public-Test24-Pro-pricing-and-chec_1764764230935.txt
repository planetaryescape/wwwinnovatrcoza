Context

This is NOT inside the member portal. It is part of the main site under Pricing.

From the pricing page, a user clicks on Test24 Pro and lands on a dedicated checkout page or modal, something like /checkout/pro-members or a similar route.

At the top of the site there is a logged in header state that shows the user name and their membership tier (e.g. a GOLD pill) when a member is logged in.

The Test24 Pro checkout already supports:

A standard Test24 Pro price

A lower member price (about R5 000 less)

An Entry Membership line item that can be added to the order

The current UI uses a simple checkbox like “I already have an active Entry Plan membership” but it does not actually check the real membership status from the logged in user.

Goal

On the public Test24 Pro checkout page, automatically detect if the current logged in user has an active Entry Membership.

If they ARE a member:

Apply the member Test24 Pro price automatically

Update the copy so the page clearly shows that member pricing is applied

Do not try to sell them membership again

If they are NOT a member (or not logged in):

Show the standard Test24 Pro price by default

Promote the benefit of adding an Entry Membership to unlock member pricing

Let them toggle whether to add the Entry Membership to this order

Do all of this on the public pricing / checkout page, without changing any logic inside the member portal.

Do not move this page to a different route. Do not change any other products or flows.

company.membershipTier (for example Starter, Growth, Scale, Entry)

or company.hasEntryMembership

or a membershipStatus or isMember flag
