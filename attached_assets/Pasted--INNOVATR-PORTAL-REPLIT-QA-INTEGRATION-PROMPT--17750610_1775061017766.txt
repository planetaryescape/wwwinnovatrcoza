
INNOVATR PORTAL — REPLIT QA & INTEGRATION PROMPT
=================================================
Paste this entire prompt into your Replit AI agent (or as a comment block at the top
of your main server file) to run a full end-to-end QA check on the portal.
=================================================


You are a senior full-stack QA engineer reviewing the Innovatr Autonomous Intelligence
Portal — a market research intelligence platform built on Replit. The portal ingests
research PDF reports and trend reports, and uses them to power AI-driven client insights,
strategic recommendations, and autonomous analysis across three core phases:
Explore, Test, and Act.

Your job is to systematically test every feature, surface every broken integration,
identify any gaps between the intended design and the current implementation, and
produce a clear fix-list ordered by priority.

Work through each section below in sequence. For each item, test it, report the result
(PASS / FAIL / PARTIAL), and if it fails, state exactly what needs to be fixed and how.

SECTION 1 — PORTAL SHELL & NAVIGATION
═══════════════════════════════════════════════════════════════════

1.1  Does the portal load without console errors? Check the browser console for
     any 404s, uncaught exceptions, or failed network requests on first load.

1.2  Sidebar navigation — verify all links work:
     · Dashboard loads the main dashboard view
     · Clicking "1 Explore" opens the Explore modal/panel
     · Clicking "2 Test" opens the Test modal/panel
     · Clicking "3 Act" opens the Act modal/panel
     · Clicking "Company" opens the Health Centre
     · Admin link routes correctly (or is hidden for non-admin users)
     · Settings opens the settings panel
     · Log out terminates the session and redirects to login

1.3  Topbar phase tabs (1 Explore › 2 Test › 3 Act) — verify each opens the
     correct phase panel. Verify the active tab is visually highlighted correctly.

1.4  Search bar (⌘K / Ctrl+K) — verify the keyboard shortcut opens a search
     interface. Verify the search queries across studies, reports, and trend signals.
     Verify results are clickable and navigate to the correct content.

1.5  Modal close behaviour — verify:
     · ✕ button closes the modal
     · Clicking the backdrop (outside the panel) closes the modal
     · Pressing Escape closes the modal
     · Closing a modal returns focus to the underlying dashboard correctly

1.6  Responsive layout — verify the portal renders correctly at:
     · 1440px wide (standard desktop)
     · 1280px wide (laptop)
     · 1024px wide (small laptop/tablet)
     · Report any layout breaks, overflow, or hidden content

SECTION 3 — PDF & REPORT INGESTION PIPELINE
═══════════════════════════════════════════════════════════════════

This is the core intelligence layer. Test it thoroughly.

3.1  PDF upload (admin side) — verify that:
     · An admin can upload a new research PDF report via the Admin panel
     · The upload is accepted and stored (check your storage bucket or database)
     · The report appears in the Trends & Insights library after upload
     · The report metadata (title, category, industry, access level, published date)
       is saved correctly

3.2  PDF text extraction — verify that:
     · After upload, the PDF's text content is successfully extracted
     · The extraction handles multi-page documents correctly
     · Scanned/image-based PDFs either extract via OCR or surface a clear error
     · The extracted text is stored in the database associated with the report record

3.3  Vector embedding — verify that:
     · After text extraction, the content is chunked and embedded into your
       vector database (MongoDB Atlas Vector Search or Pinecone)
     · The embedding process completes without errors
     · The report is searchable via semantic query within 60 seconds of upload
     · Check the embedding count in your vector DB matches the expected chunk count

3.4  Trend report visibility — verify that:
     · Reports marked "Live" appear in the Explore → Trends & Insights tab
     · Reports marked "Draft" do NOT appear to clients
     · Access level filters work: Growth-only reports are hidden from Starter users
     · Category filters (Insights / Launch / IRL / Inside) filter correctly
     · Industry filters filter correctly

3.5  Report reading — verify that:
     · Clicking a report card opens the report (PDF viewer or detail page)
     · The PDF renders correctly without broken images or missing pages
     · The reading time estimate is displayed

SECTION 12 — AI QUALITY & GROUNDING CHECKS
═══════════════════════════════════════════════════════════════════

These are the most important tests. The portal's value depends on the AI
responses being grounded in the actual ingested data — not hallucinated.

12.1 Grounding test — upload a PDF report that contains a very specific,
     unusual data point (e.g. "68.3% of LSM 7 consumers in Limpopo prefer
     rooibos-infused RTD drinks in the 250ml format"). Then ask the Research
     Assistant: "What does the data say about rooibos RTD preferences?"
     PASS: The AI returns that specific figure and cites the source document.
     FAIL: The AI gives a generic answer or invents different numbers.

12.2 Out-of-scope test — ask the Research Assistant: "What is the current
     share price of Unilever?"
     PASS: The AI says it only has access to Innovatr's research corpus and
     cannot answer general market or financial questions.
     FAIL: The AI answers using general knowledge.

12.3 Cross-study synthesis test — with at least 3 studies uploaded, ask
     the Act Planning Assistant: "Which study shows the biggest commitment gap?"
     PASS: The AI correctly identifies the study with the lowest Commitment
     score relative to its Idea score, citing the actual percentage figures.
     FAIL: The AI names the wrong study or invents figures.

12.4 Recency test — upload a new report after the session has started.
     Ask the Explore Consultant about the topic of the new report.
     PASS: The AI references content from the newly uploaded report.
     FAIL: The AI says it has no data on the topic (indicates re-indexing
     is not running on new uploads, or has a significant delay).

12.5 Strategic recommendation quality — after running a synthetic Sandbox
     test on a concept, ask the AI: "What should I do with these sandbox results?"
     PASS: The AI gives a specific recommendation that references the intent
     score from the sandbox run and suggests a logical next step
     (e.g. "At 73% intent you should proceed to a Test24 Basic study to
     validate with real respondents before investing in production.")
     FAIL: Generic advice with no reference to the actual sandbox data.

SECTION 13 — PERFORMANCE & RELIABILITY
═══════════════════════════════════════════════════════════════════

13.1 Page load time — the dashboard should fully load (including user data,
     credits, phase preview cards) within 3 seconds on a standard connection.
     Report actual load time.

13.2 AI response latency — chat responses should begin streaming within
     5 seconds of sending. If responses take longer than 10 seconds to start,
     flag this as a UX issue requiring a loading indicator or streaming implementation.

13.3 PDF ingestion time — a 20-page PDF should be uploaded, extracted,
     embedded, and searchable within 60 seconds. Test and report actual time.

13.4 Concurrent session test — open the portal in two different browser
     windows logged in as the same user. Verify that:
     · Credit changes in one window reflect in the other within 30 seconds
     · A brief submitted in one window appears in Studies in the other window

13.5 Error handling — simulate the following failure conditions and verify
     the portal handles each gracefully (no crashes, clear user-facing messages):
     · Upload a corrupted PDF → should show "File could not be processed"
     · Submit a brief with 0 credits → should show "You have no credits remaining"
     · Disconnect internet mid-chat → should show a reconnection message
     · API rate limit exceeded → should show "AI is busy, please try again"
