1. Core email architecture
Work in server/emails/email-service.ts (or the existing email service file).
1.1 Base layout
Create a base layout helper:
function renderBaseEmail(options: {
  title: string;
  greetingName?: string;  // first name, optional
  bodyHtml: string;       // main body paragraphs/markup
  buttonLabel?: string;
  buttonUrl?: string;
  showButton?: boolean;
  footerNote?: string;
}): string

Requirements:


White background, max width ~600px, centred container.


Use system fonts: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif.


Simple, elegant layout similar in feel to a HubSpot mailer but no gradients:


White body


Dark text


One primary button colour reusing the existing Innovatr button colour from the web app (do not introduce new brand colours; pull from existing styles or hardcode the same purple already used in web buttons).




Greeting:


If greetingName is provided, use Hi {{greetingName}},


Else use Hi there,




Button:


Rounded corners, solid brand colour background, white text.


Only render if buttonLabel and buttonUrl are provided.




Under the button, when relevant, support an optional “if the button does not work, copy this link” section.


Footer (same on every email):
This is an automated message from Innovatr.
If you have any questions, contact us at hello@innovatr.co.za.
© 2025 Innovatr. All rights reserved.
