Create a BriefSubmission type with at least:

createdAt – ISO timestamp

updatedAt – ISO timestamp

submittedByName – full name from the form

submittedByEmail – email from the form

companyName – company / brand they entered

studyType – e.g. Test24 Basic, Test24 Pro or other

numIdeas – number of ideas / concepts to test

researchObjective – text field

region / regions – array of strings

age / ages – array of strings

gender / genders – array of strings

income / incomes – array of strings

industry – string

competitors – array of strings

status – default "new" (later can be "in_progress", "completed" etc)

Make sure the entire payload from the front-end form is captured and stored. If some fields are optional, allow null / empty values but still save the record.
