3. What I want you to find

For the entire codebase, please:

A. Authentication and accounts

Any remaining mock auth or “fake login” paths

Flows where a user can bypass login requirements

Inconsistent use of email / id to identify a user

Places where login state can get out of sync between frontend and backend

Reset-password edge cases (invalid tokens, expired links, no user found etc.)

B. Credits and payments

Where credits are not tied to a real user account

Places where credits can go negative or not deduct correctly

Any mismatch between:

credits shown in UI

credits stored in backend

credits actually used for a study

Inconsistencies between Pay Online, Invoice Me, and Use My Credits

Flows where a user could submit a study without actually paying or using credits

C. Brief submission

Any place where brief data or attachments are not stored reliably

20MB files

multiple files

unsupported file types

D. My Research / Past Research

Cases where a user’s reports or briefs do not appear correctly

Issues with report download links or missing files

Conflicts between “Past Research” and “My Research” naming or routing

E. Admin area

Pages or tabs that reference removed / legacy data

Broken routes (404s) inside the portal

G. General robustness

Unhandled promise rejections

Console errors in browser

API endpoints that do not handle bad input
