PROJECT: Build Innovatr Consult (landing page + 4 pillar sub-pages) with a sleek, futuristic, tech-forward feel, plus a simple admin CMS to manage pillar content and anonymised case studies.

CONTEXT
Innovatr has an existing main marketing site (with hero and CTAs like “See Test24” and “See Membership Plans”). We are adding a premium “Consult” offering that sits above Test24. Consult is strategic, phased, decision-grade work for senior stakeholders (CEOs, founders, MDs, brand leaders). This page is not salesy; it’s authoritative and confident, with minimal CTAs. It should demonstrate additional value beyond basic research: longer engagements, multi-phase programmes, and deeper partnership work (like with Rain, Discovery, DGB), but without revealing confidential details.

STYLE + INSPIRATION (use as direction, not copy-paste)
We want a blend of:
1) Gladeye home page vibe: bold, striking landing that invites exploration, then deeper sections once you scroll.
2) Special Offer: modern, minimal, clean, strong typography, lots of whitespace, very intentional layout.
3) Cuberto / Instrument: premium motion, polished transitions, scroll interactions that feel high-end, not gimmicky.
4) Granny & Smith: strategic consulting energy and premium credibility.
5) Wodniack / Antinomy / Feelslike: micro-interactions, tasteful motion, unique, techy but professional.

DESIGN PRINCIPLES
- Consult should feel more elevated than the main Innovatr pages: darker, tech-forward, sleek, “AI-ish”, futuristic.
- Use restrained motion and high performance. Motion should guide meaning and progression, not distract.
- Anonymous credibility: show logos/brand names already approved on Innovatr’s main site, but do not reveal client-specific findings or confidential work.
- Minimal CTAs: 1 primary CTA at the bottom and optional subtle CTA in the hero. No repeated loud buttons.
- Mobile-first: the experience must work beautifully on mobile. On mobile, simplify heavy scroll-pin effects and replace with stacked sections and lighter transitions.

TECH STACK (assume if not explicitly known)
- Use Next.js (App Router) or React + Vite (choose what the repo uses). Keep changes consistent with existing structure.
- Use Tailwind CSS for styling.
- Use Framer Motion for animations and micro-interactions.
- For the pinned-scroll pillar “chapter” section, optionally use GSAP ScrollTrigger if already allowed. If adding GSAP is too heavy or conflicts, recreate with Framer Motion + IntersectionObserver.
- No heavy 3D libraries unless already present. Prioritise performance.
